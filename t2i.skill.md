# t2i

文生图技能（Gemini）。根据文章内容或选中文本，调用 Gemini 图像生成 API，产出与文章节奏、情绪一致的插图、信息图、封面图。

## Skill Description For Claude

使用 Gemini 将文字内容转成图片。当用户说"生成信息图""画个图""生成封面""t2i""文生图""配图"时使用。

**前置条件**：需要在 claudian 环境变量中配置 `GEMINI_API_KEY`。

---

## Instructions

### 触发条件

用户说以下任何一种：
- `/t2i`
- `生成信息图` / `生成插图` / `配图`
- `生成封面图` / `生成题图`
- `帮我画个图`
- `文生图`
- `把这个可视化`

---

### 完整工作流

#### 第 1 步：读取文章，做「视觉选角」分析

**这是最关键的一步。不要跳过。**

读取文章后，在生成图片之前，Claude 先做视觉选角分析：

**1a. 判断文章整体调性**

从以下维度快速判断，形成一个简短的「视觉基调」：

| 维度 | 问题 | 结果影响 |
|------|------|----------|
| 情绪基调 | 是批判/忧虑？还是兴奋/乐观？还是冷静分析？ | 影响图的色温、对比度、构图紧张感 |
| 内容密度 | 数据密集还是叙事散文？ | 影响图的信息量和留白策略 |
| 读者预设 | 专业从业者还是普通读者？ | 影响图的技术感 vs 亲近感 |
| 叙事节奏 | 全文一个节奏，还是有高潮低谷？ | 影响多图时的风格变化策略 |

**1b. 识别自然插图位置**

不要在每个 H2 后面机械插图。找以下类型的位置：

- **概念转折点**：文章从"是什么"转向"为什么"的地方，一张图能帮读者完成认知切换
- **数据/逻辑密集段之后**：密集文字后的视觉喘息，图不需要有信息量，给眼睛休息即可
- **核心论点处**：文章最想让读者记住的那个结论，用图锚定
- **情绪高峰**：叙事类文章的情绪顶点，用图强化而不是解释

**明确不插图的地方**：
- 连续对话段落中间（打断节奏）
- 纯过渡段（"接下来我们讨论……"）
- 结尾总结段（除非是封面型收尾）

**1c. 对每个插图位置写创意指令**

这是视觉选角的核心输出。对每个确定要插图的位置，写一段创意指令——**不是文章内容的复述，而是视觉翻译**。

**视觉锤 + 文字钉原则**：每张图必须有可读文字。视觉隐喻是锤（抓注意力），文字标签是钉（锚定意义）。纯视觉图失忆，纯文字图无力。

创意指令**必须包含以下五项**：

1. **视觉隐喻**：一个具体的画面，不是概念描述
2. **图中文字**（必填）：明确列出图中应出现的文字——标题、标签、关键短语、数字。这是 Claude 的职责，style JSON 只管怎么渲染，不管写什么
3. **图文布局**：文字和视觉元素的空间关系（文字压在图上？旁注？分区？文字居左视觉居右？）
4. **情绪注脚**：一个形容词告诉模型整体感受（压迫/轻盈/冷静/紧张）
5. **明确排除**：不要什么，防止模型发散

好的创意指令示例：
```
视觉：一个俯视岔路口，左侧道路、右侧道路分叉延伸至远处消失点。
图中文字：左侧标注「结构化」，右侧标注「去结构化」，路口中央标注「你在这里」。
图文布局：文字作为路牌贴在路面，与视觉融为一体；底部留白区域放说明文字「两种路径，取决于读者类型」。
情绪：冷静，轻微压迫感。
排除：无人物，无色彩，灰度。
```

坏的创意指令（没有文字 = 钉没打进去）：
```
"一个分叉的道路，表示两种选择，极简灰度风格"
```
→ 没有指定图中文字，模型要么不加文字，要么乱加文字

另一种坏的创意指令（文字浮在图上，没有布局关系）：
```
"分叉道路图，加上文字'结构化'和'去结构化'"
```
→ 没有说文字放哪、怎么和视觉结合

**图中文字的来源原则**：
- 从文章中提取，不要凭空发明
- 优先选：核心概念名、文章标题、关键结论短句（8 字以内效果最好）
- 避免：长句、完整段落（图里放不下，读不了）
- 数字/数据特别有力：「3 种路径」「提升 40%」比文字描述更抓眼

#### 第 2 步：确认风格

视觉选角完成后，确认 style：

```bash
python3 ~/.claude/skills/wechat-content-skills/scripts/t2i.py --list-styles
```

**风格 ≠ 内容**。风格是视觉语法（颜色规则、排版规则），内容指令是第 1 步写的创意指令。

默认映射：
- 分析/科技/逻辑类文章 → `infographic`
- 文章封面/题图 → `cover`
- 如用户有自定义风格，优先用

---

#### 第 3 步：生成图片

**优先用 `--prompt` 传创意指令**，而不是 `--input`/`--content`（那样只是把原文截段扔给模型）：

```bash
# 推荐：Claude 写创意指令
python3 ~/.claude/skills/wechat-content-skills/scripts/t2i.py \
  --prompt "一个悬停在岔路口上方的俯视视角：左侧道路标注'结构化'，右侧标注'去结构化'，用虚线表示不确定性。灰度，极简，无人物。" \
  --style infographic \
  --output "."

# 快速模式：直接扔原文（适合简单场景或快速预览）
python3 ~/.claude/skills/wechat-content-skills/scripts/t2i.py \
  --content "原文关键段落" \
  --style infographic \
  --output "."

# Pro 模型：复杂信息图、文字精度要求高时
python3 ~/.claude/skills/wechat-content-skills/scripts/t2i.py \
  --prompt "创意指令" \
  --style infographic \
  --model pro
```

多张图时，**顺序执行**，不并发（Gemini API 有速率限制）。

---

#### 第 4 步：展示结果

每张图生成后：
1. 用 `Read` 工具读取图片展示给用户
2. 告知位置和建议插入点：`第二节「读者决定了结构化策略」之后 → ![[attachments/infographic_读者决定了结构化策略.png]]`
3. 如果多张图，最后整理一个插入建议列表

---

#### 第 5 步：迭代

| 用户反馈 | 处理方式 |
|----------|----------|
| "情绪不对，太冷了" | 修改创意指令的情绪注脚重新生成 |
| "信息太多太乱" | 创意指令里加"极简，只保留核心隐喻，大量留白" |
| "换个风格" | 换 `--style` |
| "文字不清楚" | 加 `--model pro`；或修改 vault 里的 prompt 文件 |
| "这个位置不对" | 重新讨论插图位置，按新位置的文本重写创意指令 |

### Prompt 管理

#### 两层设计：视觉语法 vs 内容指令

| 层 | 存储位置 | 职责 | 谁写 |
|----|----------|------|------|
| 视觉语法 | `prompts/*.json` 的 `system` 字段 | 颜色规则、字体规则、构图约束 | 人工维护 |
| 内容指令 | 每次由 Claude 生成 | 画什么、什么情绪、什么构图 | Claude 实时生成 |

#### 内置 Prompt 位置

```
~/.claude/skills/wechat-content-skills/prompts/
  infographic.json   # 技术信息图（高对比度黑白，文档级）
  cover.json         # 公众号封面图（16:9，编辑风格）
```

每个文件结构：
```json
{
  "name": "显示名称",
  "description": "用途描述",
  "tags": ["tag1", "tag2"],
  "aspect_ratio": "1:1",
  "system": "视觉语法约束..."
}
```

#### 添加/覆盖 Prompt（在 Obsidian 里编辑）

在 vault 根目录创建 `t2i-prompts/` 文件夹：

```
{vault-root}/
  t2i-prompts/
    infographic.json   ← 覆盖内置（可在 Obsidian 里直接改）
    dark-poster.json   ← 新增自定义风格
```

**vault 优先级高于内置**，同名时 vault 版本生效。

### 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--prompt` / `-p` | Claude 生成的创意指令（优先级最高） | — |
| `--content` / `-c` | 直接传原文段落（快速/简单场景） | — |
| `--input` / `-i` | Markdown 文件路径 | — |
| `--style` / `-s` | 视觉语法风格 ID | `infographic` |
| `--output` / `-o` | 输出路径或目录 | vault attachments 或 /tmp |
| `--model` / `-m` | `flash`（快）或 `pro`（精） | `flash` |
| `--list-styles` | 列出所有风格 | — |

**优先级**：`--prompt` > `--content` > `--input`

---

### 图片命名规则

生成的图片使用自然语言命名，便于识别和管理：

- **格式**：`{style_id}_{描述性名称}.png`
- **来源**：
  - `--prompt`：从创意指令的前50个字符提取
  - `--content`：从内容文本的前50个字符提取
  - `--input`：从文件名提取（去掉扩展名，转换分隔符）
- **处理**：自动清理特殊字符，限制长度为30字符，转换为下划线分隔
- **示例**：
  - `infographic_读者决定了结构化策略.png`
  - `cover_文章标题示例.png`
  - `infographic_20240325_143022.png`（fallback时间戳）

### 环境变量配置

在 claudian 的设置 → Environment Variables 中添加：

```
GEMINI_API_KEY = your-api-key-here
```

脚本从 `os.environ.get("GEMINI_API_KEY")` 读取，不落盘。
